• Pricing model: on demand, reserved instance(contract), on spot(bid), dedicated host(licence)
  • Read whitepages about EC2 vs lambda
  • Uses EBS(Elastic Block Storage) running on VPC(virtual private cloud) as storage
  • Use Putty SSH to connect, allow port 22 is a must, 80,443,3389 are optional
  • EBS is distributed in different AZs, like S3
  • EBS has the following types
    • General purposed SSD, ~100, max 3000 IOPS
    • Provisioned IOPS SSD, >10000 IOPS
    • Throughout optimised HDD, connot be boost disk
    • Cold HDD, low cost, for IA
    • Magnetic, bootable cheap, slow, rarely used
  • It's still a server, not serveless
  • EC2 can be assigned with roles with different access, so credentials don't need be stored
  • Roles can be attached on the go, but this is a recent feature
  • Roles are universal, not reginal
  • Apache is used to host website on EC2, yum install httpd -y
  • You can set up auto scalling and it will launch instances when needed
